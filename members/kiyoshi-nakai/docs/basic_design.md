## 0-2. グループ共通部品との接続設計

> ※ グループで決めた「共通部品」と、あなたの個人設計の接点を明確にします。

| 項目 | 内容 |
|:--|:--|
| グループ共通部品（名称） | LCDパネル（LCM1602A） |
| 共通部品が担う役割 | キャラクター位置と色状態を表示する出力インターフェース |
| 接続するArduinoピン（予定） | D7（RS）, D8（E）, D9-D12（データ）, VO（可変抵抗）, RW（GND固定）, 5V, GND, A/K（バックライト） |
| 個人の設計と共通部品が協調する箇所 | ジョイスティック入力で座標更新、IR受信入力で色状態を変更しLCD表示へ反映する |
| グループ内で統一すべき仕様 | LCD配線割り当て（RS/E/D4-D7, VO, RW, A/K）、表示更新周期（100ms）、座標原点（左上）、色切替コード割り当て |

> グループメンバー全員が同じピン番号・同じ閾値の単位を使っているか、
> グループレビューで確認し合ってください。

## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

**記入例：**
```
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理    ├──→ パッシブブザー
                      └─────────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】                 【Arduino UNO R3】            【出力】
Joystick Module ───→┐                         ┌──→ LCM1602A（パラレル）
IR Receiver Module →┤─ 入力読取り・状態管理・描画 ┤
（Remote Control）   └─────────────────────────┘
```

補足説明（図で表現しきれない場合）：

ジョイスティックのX/Y軸を方向入力に変換し、LCD上のキャラクター座標を更新する。Remote Controlのボタン入力はIR Receiver Module経由で受信し、表示色を切り替える。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（LCD初期化完了）
[待機中] ──（ジョイスティック入力あり）──→ [移動中] ──（端到達または一定時間入力なし）──→ [待機中]
   │                                      │
   └──（Remote Control入力）──→ [色変更中] ←──（Remote Control入力）──┘
                              ↓（色反映完了）
                            [待機中 / 移動中に復帰]


```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| 初期化中 | LCD初期化、カスタム文字登録、初期座標セット | LCDに "Init..." を1回表示 |
| 待機中 | 100ms周期で入力監視、入力があれば移動中へ遷移 | キャラクターを現在座標に静止表示（現在色を反映） |
| 移動中 | 100ms周期で方向判定し、座標を1マスずつ更新 | キャラクター位置を更新表示（現在色を反映） |
| 色変更中 | IRコードを解釈して colorMode を更新し表示へ反映する | 色状態を更新したキャラクターを再描画 |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「状態が固定されたまま戻れない」ケースがないか確認してください。

### 2-2. 機能一覧・関数一覧（役割の定義）

> ※ プログラムを「処理の部品（関数）」に分割します。
> **ここではコードを書かず、「何をする関数か」だけ日本語で書いてください。**
> 具体的な実装は詳細設計書（detailed_design.md）で行います。

> [!TIP]
> **単一責任の原則（1関数 = 1つの仕事）**
> 「機能名」欄に requirements.md の機能名（必須・追加）を書き、それを実現する関数名を対応させてください。
> 入出力の読み取り（readButton 等）や制御（updateOutput 等）は複数機能から使われる共通処理として「共通」と書いてOKです。

| 機能ID | 機能名 | 関数名 | 担う「1つの仕事」 | 主な引数 | 戻り値 | 呼び出す場所 |
|:--|:--|:--|:--|:--|:--|:--|
| — | 初期化 | `setup()` | ピンモード設定・ライブラリ初期化・起動確認 | なし | なし | 起動時1回 |
| — | （共通）待機・制御 | `loop()` | 状態に応じて各関数を呼び出すメインループ | なし | なし | 常時 |
| — | （共通）ボタン読出 | `readButton()` | チャタリング処理済みのボタン状態を返す | なし | bool | loop()内 |
| — | （共通）ジョイスティック読出 | `readJoystick()` | ジョイスティックX/Yの生値を取得する | なし | 構造体/配列 | loop()内 |
| — | （共通）IR受信読出 | `readIR()` | Remote Controlの赤外線コードを取得する | なし | unsigned long | loop()内 |
| — | （共通）LCD更新 | `updateDisplay()` | 現在のstateと座標に応じてLCDを更新する | int state | なし | loop()内 |
| F01 | 必須機能①: ジョイスティックX連動で左右移動 | `updateHorizontalMove()` | X軸値から左右移動を判定して座標更新 | int joyX | なし | loop()内 |
| F02 | 必須機能②: ジョイスティックY連動で上下移動 | `updateVerticalMove()` | Y軸値から上下移動を判定して座標更新 | int joyY | なし | loop()内 |
| F03 | 必須機能③: Remote Control入力で表示色切替 | `updateColorByRemote()` | IRコードに応じて色モードを切り替える | unsigned long irCode | なし | loop()内 |
| C03 | （共通）キャラクター描画 | `drawPacman()` | 現在座標と色モードでキャラクターを描画する | uint8_t x, uint8_t y | なし | loop()内 |
| A01 | 追加機能①: キャラクター切替 | `toggleCharacter()` | SW押下で表示キャラを切り替える | bool pressed | なし | loop()内 |

> [!CAUTION]
> loop() の中で全部書こうとしていませんか？
> 機能ごとに関数を分けることで、グループレビューや単体テストがしやすくなります。

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| ジョイスティック入力読み取り | 100ms | **millis** | 表示更新と並行しつつ操作遅延を抑えるため |
| LCD表示更新 | 100ms | **millis** | 入力に追従しながら他処理を止めないため |
| IRコード受信確認 | 50ms | **millis** | リモコン入力に素早く反応するため |
| SWデバウンス判定 | 50ms | **millis** | チャタリングを抑制するため |
| 起動メッセージ表示 | 300ms | **delay** | 起動時1回のみで他処理停止の影響がないため |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もジョイスティック読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針

| 異常ケース | 発生条件 | 検知方法 | 対応方針 |
|:--|:--|:--|:--|
| チャタリング | ボタンを押したとき | 前回入力から50ms未満の入力を検出 | デバウンス処理でスキップ |
| ジョイスティック異常値 | 断線や接触不良で異常値が出たとき | A0/A1が0付近または1023付近で固定される | 一定回数連続した異常値を無視して中央値扱いにする |
| アナログノイズ | アナログ入力を読んだとき | 閾値付近で値が揺れる | 閾値に±10〜20のバッファを設ける |
| IR誤受信 | 照明ノイズや無関係な赤外線を受けたとき | 未登録のIRコードを検出 | 登録済みコードのみ有効化し、それ以外は破棄 |
| LCD表示失敗 | LCD初期化や配線不良があるとき | 起動時の"Init..."表示確認で検知 | エラー文をシリアル出力し待機状態に移行 |
| 座標範囲外更新 | 連続入力時に境界を超えるとき | 更新前に x:0-15, y:0-1 をチェック | 範囲外は切り捨てて境界値に固定 |

## 7. AIレビュー記録

> グループレビューの前に必ず実施してください。

### Q1: 設計の妥当性確認
